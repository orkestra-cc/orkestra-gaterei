# Each line is a reconciliation diagnostic the scanner flagged at the time
# the CI gate was introduced (Phase 5.1, 2026-04-20). CI fails on any new
# entry NOT listed here, so the gate catches future drift even while the
# backlog is worked off.
#
# Format: <category>:<key>
#
# Categories that may fail CI:
#   permission.used.undeclared     — route requires a permission not in any Permissions() catalog
#   permission.declared.unused     — Permissions() catalog entry never referenced by a Require* call
#   capability.used.undeclared     — route requires a capability not in any Capabilities() catalog
#
# Warn/info categories (never fail CI today):
#   capability.declared.unused     — published capability not yet consumed by a RequireCapability
#   gate.dynamic_key               — Require* called with a non-literal first argument
#
# To reconcile an entry: either remove the declaration from the owning module's
# Permissions()/Capabilities(), or wire a route that actually uses the key.
# Then delete the corresponding line from this file.
#
# DO NOT add new entries by hand. If a diagnostic is legitimate (e.g. a
# permission published ahead of the feature that will consume it), reconcile
# it at the call site rather than baselining.
#
# Regenerate after legitimate drift is cleaned up:
#   go run ./tools/policycoverage/cmd/policycoverage ./internal/...
