# Operating Orkestra

Once you have Orkestra running, the operator-side surface is mostly:

1. **Deployment topology** — single binary, AI sidecar, or fully split.
2. **Module configuration** — env vars, encrypted secrets, per-environment profiles (sandbox / production).
3. **Integrations** — SMTP for email, OAuth providers, Stripe for payments, Gotenberg for PDF generation.
4. **Observability** — health checks, audit logs, structured logging.
